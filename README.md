# 四柱推命 鑑定書（八字）

生年月日時、性別、標準時経度、地方時差などから四柱（八字）を表示する、依存関係のない静的Webアプリです。

`index.html` をブラウザで開くか、次のようにローカルサーバーを起動してください。

```powershell
python -m http.server 4173
```

現在、入力フォーム・八字・大運（順逆、立運年齢、10運）・今年を中心とした前後5年の年運・原命式の五行サークルに加え、原命式＋選択年の大運＋年運を十二文字で表示する「六柱推命モード」を表示します。

結果画面の「鑑定書PDF出力モード」では、鑑定年を0〜120歳から選び、原命式・原命式の五行バランス・指定年の十二文字・その五行バランスをA4横1枚でプレビューできます。用途は、現在のシンプルな「お客さまお渡し用」と、五行点数・柱別の五行変化根拠・成立関係を併記する「講座生添削用」から選べます。「PDFとして保存」から印刷画面を開き、送信先に「PDFに保存」を選択してください。

西暦1年から9999年まで入力できます。3桁以下の年は、先頭に0を付けて入力します（例：845年8月1日 → `0845-08-01`）。

出生時刻には、出生地の地方時差と添付資料の日別均時差表を符号どおり加減します。また、日本標準時を選択した1948〜1951年のサマータイム実施期間は、入力時刻から60分を自動で戻してから時柱を判定します。

## 現在の五行ルール

- 木：甲・乙・寅・卯（`#7ed957`）
- 火：丙・丁・巳・午（`#f42e2e`）
- 土：戊・己・丑・辰・未・戌（`#ffae5c`）
- 金：庚・辛・申・酉（`#898888`）
- 水：壬・癸・子・亥（`#38b6ff`）
- 月支・大運支は3枠、時支・日支・年支・年運支は1枠として、十二支ごとの確定枠と変化可能枠を管理する。
- 無作用の丑・辰・戌は、五行変化が成立するまで五行カラーを表示しない（月支の丑は水2点確定）。
- 火土同根：天干の戊・己（干合による土化を含む）がある場合、巳・午へ火と同量の土を加える。未は資料に従った火・土の確定枠を持つ。
- 地支変化の優先順位：亡神 → 方合 → 会局 → 疑似局 → 支合 → 冲 → 半方合 → 半会 → 土化。
- 完成形が成立した場合は下位の未完成形を適用しない。半方合・半会は距離により1、0.5、0.25点で作用する。
- 原命式では、複数の五行変化が同じ地支へ重なっても、その地支の変化可能枠を越えない。枠を越える場合は、各五行の作用量を同じ比率で按分する（例：1枠へ水1・木1が重なる場合は水0.5・木0.5）。
- 原命式では、変える側が起こせる五行変化の合計を、その支が実際に持つ季節五行の量までとし、優先順位と距離順に使用量を差し引く（例：月支未の火化・土化力は確定火2から開始する）。
- 火土同根の火と土は「1枠の表裏」として扱い、火1・土1でも使用枠は合計1枠と数える。
- 支合が成立した組み合わせは、その相手同士について下位変化から除外する。冲は抜根条件だけを判定し、抜根する・しないに関わらず、すべての支をそのまま半方合 → 半会 → 土化へ流す。
- 半方合は同じ方合グループ内で、変える力を持つ支から相手の変化可能枠へ作用させる。半方合を半会・土化より先に処理し、実際に作用した量を変える側の残り変化力から差し引く。
- 巳・午から丑・辰への相互作用による土化は、変える側に残り変化力がある場合だけ成立し、成立量を残量から差し引かない。巳から戌、未から丑への土化は作用量を残り変化力から差し引く。
- 支合は資料で変化条件と変化量が明記された組み合わせを適用する。冲は天干・地支合計が5対1となる抜根のみ数値へ反映する。
- 干合は隣接・妬合なし・化気五行の地支4点以上・漏財官3点以下・日主の追加条件を満たした場合のみ化気する。
- 大運・年運は原命式から下流へ影響を受けるが、大運・年運から原命式へは逆流させない。
- 同じ五行が上下左右で接する原命式は一つの連結枠にまとめる。大運・年運は各1本の中だけを連結する。
- 原命式では、亡神・方合・会局・疑似局・支合・冲を地支へ、干合・近貼を天干へ角丸スタンプで表示する。隣接する支合・冲・干合は2文字の中央へ1個だけ表示する。支合・干合は五行変化の成否を問わず、冲は抜根の成否を問わず表示する。
- 成立条件は一括表示ではなく、時支・日支・月支・年支ごとに、作用元・作用先・変化した五行量を表示する。
- 六柱推命モードは0歳から120歳までの年運を「西暦年／年齢／大運干支」のプルダウンで選択し、その年の大運2文字・年運2文字を原命式8文字へ加える。時柱・日柱・月柱・年柱・大運・年運の6本を同じ五行カラーと関係スタンプで表示する。
- 立運前の年は大運を表示・集計せず、原命式8文字＋年運2文字の十文字で五行バランスを計算する。プルダウンの3項目目は大運干支の代わりに「立運前」と表示する。
- 干合・支合などの関係マークは五行変化後の色で表示する。六柱推命モードの甲己干合は両方を土へ変化させ、未午支合は火色で表示する。冲は単一の変化五行を持たないためグレー表示を維持する。
- 六柱推命モードでは、大運干・大運支は原命式と年運の全柱に、年運干・年運支は原命式と大運の全柱に隣接するものとして扱う。これらの作用距離は画面上の並びに関係なく1とする。
- 六柱推命モードの表示では原命式4柱と大運・年運2柱の間に余白を設け、同じ五行でもこの表示境界をまたいで枠線を連結しない。判定上の隣接関係は維持する。
- 天干・地支の関係候補は原命式 → 大運 → 年運の順で確定する。辛－丙－辛、丙－辛－丙など同じ干で対象干を挟む関係は「妬合」とし、3干すべてを化気対象から外して水色の妬合マークを表示する。先に妬合へ使われた干は後続候補へ重複使用しない。
- 半方合を半会より先に確定し、半方合後に残った変化可能枠だけを半会へ渡す。
- 亥子・卯寅・巳午・申酉の半方合は、互いの五行を変えず、双方の変化エネルギーも消費しない。ただし同じ受け枠を半会と取り合う場合は、エネルギーを消費せずに半方合の五行を先に確定する。旺支の午は残した火化力を次の半会へ使える。
- 土化は隣接関係に限定する。巳・午×丑・辰だけは相互に土を持つ。午×戌は土化ではなく半会として午は火のまま、戌へ火・土を同量加える。巳×戌は巳を火のまま保ち、戌だけへ土を加える。半会など上位変化で受け枠が確定済みなら、後続の土化は入らない。
- 十二文字の五行サークルでは、大運支を3点枠、年運支を1点枠として集計し、時支・日支・月支・年支・大運支・年運支の成立説明を表示する。

資料に具体的な減点量がない支合・冲の「互いに弱め合う」作用、近貼の性質付加、刑・破・害は得点へ反映していません。

## Render実証版

既存の鑑定画面を、1つのRender Web Serviceからログイン必須で配信できます。実証版では共通アクセスパスワードを使用し、ログイン後は署名付き・HttpOnly・SameSite Cookieで認証状態を保持します。

### 現在の実証範囲

- 未ログイン時は鑑定画面、JavaScript、CSS、認証済みAPIを配信しない
- `/health` はRenderのヘルスチェック用として公開する
- `server.js`、テスト、環境変数などのサーバー内部ファイルは配信しない
- ログイン失敗を同一接続元ごとに制限する
- 計算結果と既存UIは変更せず、全自動テストを維持する

この第1段階では計算処理はまだ `app.js` に含まれています。未ログインの第三者からは保護されますが、ログインを許可された利用者のブラウザには配信されます。計算ロジック自体の保護は、計算エンジンをAPIへ分離する第2段階で完了します。

### ローカル確認

PowerShellで環境変数を設定して起動します。

```powershell
$env:KANTEISHO_ACCESS_PASSWORD='実証用パスワード'
$env:SESSION_SECRET='十分に長いランダム文字列'
npm start
```

ブラウザで `http://localhost:3000` を開きます。環境変数を設定しない開発環境では、ローカル確認専用パスワード `local-test-only` が使用されます。本番環境では環境変数がなければ起動しません。

### Render設定

リポジトリ直下の `render.yaml` を使うか、既存Web Serviceへ次を設定します。

- Build Command: `npm install`
- Start Command: `npm start`
- Health Check Path: `/health`
- `NODE_ENV`: `production`
- `KANTEISHO_ACCESS_PASSWORD`: 講座生へ知らせる実証用パスワード
- `SESSION_SECRET`: Renderで生成する長いランダム値
- `SESSION_HOURS`: `12`

`KANTEISHO_ACCESS_PASSWORD` と `SESSION_SECRET` はGitHubへ保存せず、Render Dashboardの Environment に登録します。

### 独自ドメイン

Renderの Custom Domains に `kanteisho.fuchilabo.com` を追加し、ドメイン管理画面のDNSへRenderが案内するCNAMEを設定します。接続確認後は Render Subdomain を無効化すると、`onrender.com` 側からのアクセスを遮断できます。
